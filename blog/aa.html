                # Vue首屏渲染，白屏，原因与解决办法

                FP（first Paint）第一帧渲染 FCP（first Content paint） 首次内容渲染 FMP（first meaning paint）首次有意义的渲染

                骨架屏在FP过程实用

                加载过程： ===> html FP --> html页面加载下来 <静态资源>

                ===> 静态资源 css js ==> 解析js ==> 生成html FCP ---> css、js静态资源加载下来 ===> ajax请求数据 FMP ---> ajax数据请求回来 --------------------------以上为白屏

                1. 路由动态加载，提取公共库

                在babel 里配置 plugin-syntax-dynamic-import ==> 路由动态加载 （配置webpack： optimization.splitChunks)

                1. ssr 预渲染 同构

                ssr => 服务端渲染 （php， jsp） 同构渲染 ==> 一套代码多端使用（vue react nuxt） vue ----> json ----> vue-server-render ----> html 预渲染 ===> 预渲染 webpack vue打包出来的项目 ---> 无头浏览器 --> 获取到预渲染的页面html内容 ----> index.html ----> CDN ---> index.html

                ssr (会有并发问题) === 请求 ===> node（执行vue项目）===> html ===> 客户端

                1. loading --> 骨架屏 --> 预渲染

                2. webpack entry

                3. 资源请求时间片的处理

                4. CDN

                5. quicklink ---> 在浏览器空闲的时候去解析预加载我可能要跳转的页面 prefetch preload preconnect

                  preload ===>

                # Object.defineProperty() 能不能监听数组？

                  function defineReactive () {
    Object.defineProperty(data, key, {
      enumerable: true,
      configurable: true,
      get () {
        console.log(`get获取的key: ${key}, value: ${value}`)
        return value
      },
      set(newVal) {
        console.log(`set=设置的key：${key}, value: ${newVal}`)
        value = newVal
      }
    })
  }

  function observe(data) {
    Object.keys(data).forEach(function(key) {
      defineReactive(data, key, data[key])
    })
  }

  var arr = ['小明', '小华', '小刚']

  observe(arr)

                # 为什么在vue2源码里重新数据的监听和操作方法？？

                不是 Object.defineProperty 监听不到数组的变更，因为对于数组的每一次操作，都可能影响其他 key 索引的变更。在多数情况下，需要去遍历数组，重新检测。
