# 渲染

  1. 当组件的 state 或 props 发生改变的时候，render 函数就会重新执行

  2. 当父组件的 render 函数执行后，对应的所有子组件的 render 函数都会执行一次

  3. react 生成虚拟DOM，虚拟DOM是js对象，在js里生成js对象进行对比操作，相比于直接对比真实DOM提升了很大的性能

  4. 渲染步骤 a. state 数据 b. JSX 模板 c. 数据 + 模板 生成虚拟DOM （虚拟DOM就是一个JS对象，用它描述真实的DOM） d. 用虚拟DOM的结构生成真实的DOM 来显示 e. state 发生变化 f. 数据 + 模板 生成新的虚拟DOM g. 比较原始虚拟DOM和新的虚拟DOM的区别，找到不同的地方 h. 直接操作真实DOM ，改变不同地方的内容

优点:

  1. 性能提升了

  2. 他使得跨端应用得以实现。 React Native

  3. 生命周期函数 a. componentWillMount 在组件即将被挂载到页面的时刻自动执行 b. componentDidMount 在组件被挂载到页面后，自动执行

c. shouldComponentUpdate 在组件被更新之前会自动执行，但需要返回一个布尔类型的结果 true 会执行 false 不会更新 d. componentWillUpdate 在组件更新之前，会自动执行，且在 shouldComponentUpdate 返回 true 后执行，如果返回 false 这个函数就不会执行 e. componentDidUpdate 在组件更新之后，会自动执行

f. componentWillReceiveProps 当一个组件从父组件接收了参数，只要父组件的 render 被重新执行后，子组件会执行该生命周期(第一次存在于父组件不会执行，已经在父组件中，才会执行) g. componentWillUnmount 当组件即将被剔除的时候，会被执行（子组件）
